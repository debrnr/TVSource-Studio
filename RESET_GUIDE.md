# TVSource Studio - 项目重置指南

## 📝 项目历史

本项目最初目标是创建一个简单的TVBox源聚合服务,但在开发过程中逐渐演变成了包含多种适配器架构的复杂系统。

### 遇到的问题

1. **过度设计**: 从简单的API聚合发展到支持XBPQ规则引擎、DRPY2 JS脚本等多种数据源类型
2. **配置混乱**: DEMO目录中的资源不断导入,导致69个数据源难以管理
3. **适配未完成**: XBPQ和DRPY2适配器框架已搭建但核心方法未实现
4. **维护困难**: 代码复杂度远超实际需求,调试和维护成本高

### 经验教训

✅ **应该做的**:
- 保持简单,优先实现核心功能(MacCMS API聚合)
- 小步迭代,每步都验证可用性
- 明确边界,区分服务端和客户端职责

❌ **不应该做的**:
- 盲目追求功能完整性
- 一次性导入大量未验证的资源
- 在未测试的情况下扩展架构

## 🔄 重置选项

### 选项1: 仅重置数据(推荐)

保留所有代码,只清空配置和数据文件,从头开始配置:

```bash
python reset_data.py
```

**适用场景**: 
- 想保留代码框架
- 希望重新配置数据源
- 不想重新安装依赖

---

### 选项2: 完全清空项目

删除所有文件,仅保留README:

```bash
python clear_project.py
```

**适用场景**:
- 想彻底重新开始
- 需要清理所有代码
- 准备从Git恢复或重新克隆

---

### 选项3: Git恢复(如果使用了版本控制)

```bash
git reset --hard HEAD
git clean -fd
```

## 💡 重新开始建议

如果决定重新开始,建议采用**极简方案**:

### 最小可行产品(MVP)

**核心功能**:
1. ✅ MacCMS API聚合(3-5个稳定源)
2. ✅ 标准MacCMS接口(`/api/maccms/vod`)
3. ✅ TVBox配置生成(`/api/tvbox/config`)
4. ✅ 简单的Web管理界面

**暂不实现**:
- ❌ XBPQ规则引擎
- ❌ DRPY2 JS脚本
- ❌ 复杂的HTML解析
- ❌ 自动源发现

**技术栈**:
- Python + Flask
- requests库(HTTP请求)
- SQLite(可选,用于缓存)

**预期效果**:
- 7个以内的数据源
- 稳定的API响应
- 易于维护和扩展

## ⚠️ 重要提醒

**在清空项目前**:
1. 确认是否需要备份任何代码或配置
2. 检查是否有未提交的Git更改
3. 记录下已成功工作的配置(如果有)

**重新开始后**:
1. 从最简单的功能开始
2. 每添加一个功能都要测试
3. 避免过早优化和过度设计
