### 2. 文件分类规则

#### test/ - 测试和临时脚本
**存放内容**：
- ✅ 开发调试脚本
- ✅ 问题诊断工具
- ✅ 一次性测试代码
- ✅ 实验性功能验证

**特点**：
- 可能随时删除或修改
- 不保证稳定性
- 主要用于开发和调试

**示例**：
- `diagnose_xxx.py` - 诊断工具
- `test_xxx.py` - 功能测试
- `check_xxx.py` - 数据检查
- `verify_xxx.py` - 验证脚本

---

#### tools/ - 核心工具脚本
**存放内容**：
- ✅ 生产环境使用的工具
- ✅ 稳定的适配器实现
- ✅ 定期运行的维护脚本

**特点**：
- 经过充分测试
- 被正式代码调用
- 需要长期维护

**示例**：
- `k1k_adapter.py` - k1k.cc适配器（被routes.py调用）
- `health_check_api_sources.py` - API源健康检查（定时任务）

### 3. Git管理策略

#### .gitignore配置

```gitignore
# Test scripts (optional to commit)
test/*.py
!test/README.md
```

**说明**：
- ❌ **默认忽略**所有 `test/*.py` 脚本
- ✅ **保留** `test/README.md` 文档
- ✅ **例外**：有价值的诊断工具可以手动添加到Git

**判断标准**：
```
是否需要提交到Git？
├─ 是 → 一次性临时测试？
│   ├─ 是 → ❌ 不提交
│   └─ 否 → 有长期价值？
│       ├─ 是 → ✅ 手动添加 git add -f test/xxx.py
│       └─ 否 → ❌ 不提交
└─ 否 → ❌ 不提交
```

### 4. 命名规范

#### 推荐命名格式

| 类型 | 前缀 | 示例 |
|------|------|------|
| 诊断工具 | `diagnose_` | `diagnose_tvbox_issue.py` |
| 功能测试 | `test_` | `test_api_response.py` |

#### 命名原则
- ✅ 使用小写字母和下划线
- ✅ 清晰描述功能
- ✅ 避免缩写（除非非常通用）
- ❌ 不要使用中文文件名

### 5. 脚本编写规范

#### 基本要求

1. **自包含**：脚本应能独立运行，无需额外参数
2. **清晰输出**：使用分隔线和emoji提高可读性
3. **错误处理**：包含try-except捕获异常
4. **中文注释**：关键步骤添加中文说明
5. **超时设置**：网络请求必须设置timeout

#### 模板示例

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
"""测试脚本简要描述

详细说明（可选）：
- 用途
- 使用方法
- 注意事项
"""

import requests
import json
import sys

print("="*80)
print("测试标题")
print("="*80)

try:
    # 1. 准备测试数据
    print("\n📡 准备测试...")
    url = "http://127.0.0.1:8080/api/xxx"
    params = {'key': 'value'}
    
    # 2. 执行测试
    print(f"🔍 测试URL: {url}")
    response = requests.get(url, params=params, timeout=10)
    
    if response.status_code != 200:
        print(f"❌ HTTP错误: {response.status_code}")
        sys.exit(1)
    
    data = response.json()
    
    # 3. 验证结果
    print(f"\n✅ 测试结果:")
    print(f"   状态码: {data.get('code')}")
    print(f"   数据量: {len(data.get('list', []))}")
    
    # 4. 详细分析
    print("\n📊 详细分析:")
    if 'class' in data:
        print(f"   ✅ class字段存在")
    else:
        print(f"   ❌ class字段缺失")
    
    print("\n" + "="*80)
    print("✅ 测试完成！")
    print("="*80)

except Exception as e:
    print(f"\n❌ 测试失败: {e}")
    import traceback
    traceback.print_exc()
    sys.exit(1)
```

## ⚠️ 注意事项

### 1. 不要混淆 test/ 和 tools/

| 维度 | test/ | tools/ |
|------|-------|--------|
| 用途 | 开发调试 | 生产使用 |
| 稳定性 | 不稳定 | 稳定 |
| 调用方式 | 手动运行 | 被正式代码调用 |
| Git管理 | 默认忽略 | 必须提交 |
| 清理频率 | 经常清理 | 长期保留 |

### 2. 敏感信息

**严禁在测试脚本中硬编码**：
- ❌ API密钥
- ❌ 数据库密码
- ❌ 服务器IP地址（使用环境变量）

**正确做法**：
```python
import os
api_key = os.getenv('API_KEY', 'default_value')
server_ip = os.getenv('SERVER_IP', '127.0.0.1')
```

### 3. 依赖管理

如果测试脚本需要额外的Python包：
1. 添加到 `config/requirements.txt`
2. 或在脚本开头注释说明依赖

```python
# 依赖: pip install xxx
import xxx
```
